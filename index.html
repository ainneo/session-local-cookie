JWT is just for Autherization which is making sure the same user that was Authenticated durning the login process has acess to a particular system. The way this is normally done is sending session ids in cookies in the browser. Everytime the client makes a request, they send that session id to the server. The server checks it's memory and looks for the user that has the matching session ID and Authorizes the user to have access. BUT JWOT does not use cookies, it uses a JSON web token.
